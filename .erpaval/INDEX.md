Lessons learned from prior ERPAVal sessions. Claude reads this at

## By category

### api-patterns/

- [DuckDB ATTACH catalog check](solutions/api-patterns/duckdb-attach-catalog-check.md) — ATTACH on a fresh path leaves a 12 KB header-only file; gate rebuild on `duckdb_tables()`, not file size.
- [DuckDB memory_limit % rejection](solutions/api-patterns/duckdb-memory-limit-percent.md) — `'70%'` raises Parser Error; resolve to MiB at apply time via `os.sysconf`.
- [DuckDB list_avg is per-row](solutions/api-patterns/duckdb-list-avg-aggregate.md) — for centroid math use `unnest+generate_subscripts+groupby+list ORDER BY pos`.
- [Bedrock shared client + pool sizing](solutions/api-patterns/bedrock-shared-client-and-pool.md) — one shared boto3 client with `max_pool_connections=max(32, 2×conc)`; default of 10 silently caps concurrency.
- [Anthropic tokenizer gap + cache minimums](solutions/api-patterns/anthropic-prompt-cache-tokenizer-gap.md) — cl100k_base overcounts Anthropic tokens ~22%; Haiku 4.5 cache min is 4096 vs Sonnet's 1024.
- [Global CRIS not in ListInferenceProfiles](solutions/api-patterns/cris-profile-not-in-list.md) — `global.<vendor>.<model>` may invoke fine even when absent from the profile list; verify by invoke, not listing.
- [ty strict mode is `rules.all = "error"`](solutions/api-patterns/ty-strict-mode-all-error.md) — ty has no `--strict` flag; the canonical strict knob is `[tool.ty.rules] all = "error"` + `error-on-warning`, minus `division-by-zero`.
- [hdbscan cp314 wheel gap](solutions/api-patterns/hdbscan-cp314-wheel-gap.md) — 0.8.42 is the lone Python 3.14 blocker in the scientific-python closure; sdist fallback is unacceptable for `uv tool install` end-users.
- [Lefthook pre-push `HEAD~` fallback](solutions/api-patterns/lefthook-push-gate-fallback.md) — `@{push}` fails with exit 128 on first-push-of-branch; keep the `|| git diff --name-only HEAD~` fallback.
- [Semgrep in CI: container, not deprecated action](solutions/api-patterns/semgrep-ci-container-not-action.md) — `returntocorp/semgrep-action` is deprecated; `semgrep ci` rejects `--config`. Run `semgrep scan` inside the `semgrep/semgrep` container.
- [CycloneDX SBOM for uv projects](solutions/api-patterns/cyclonedx-python-uv-environment.md) — cdxgen + `uv export` don't speak uv.lock; `cyclonedx-py environment .venv` is the working path.
- [Bedrock structured output: `output_config.format` ≠ Converse](solutions/api-patterns/bedrock-output-config-vs-converse.md) — two API shapes with different field paths and refusal vocabularies; stay on InvokeModel + `output_config.format` to match the existing pipeline (refusal stop_reason, prompt caching, adaptive thinking).
- [PR↔transcript binding: trailers + git notes from `prepare-commit-msg`](solutions/api-patterns/git-trailer-and-notes-binding.md) — commit-trailers + `refs/notes/transcripts` is the survivable, host-agnostic primitive; run from `prepare-commit-msg` (not `commit-msg` or `pre-commit`).
- [bandit `-f sarif` needs the `[sarif]` extra](solutions/api-patterns/bandit-sarif-formatter-extras.md) — without `bandit[sarif]`, `-f sarif` raises `ModuleNotFoundError` at runtime, not install time. Pair with `[tool.bandit] skips` aligned 1:1 with ruff's S-ignores.
- [betterleaks isn't on PyPI — Go binary, install via mise `aqua:`](solutions/api-patterns/betterleaks-not-on-pypi.md) — `uv add betterleaks` fails (not a Python package); use `aqua:betterleaks/betterleaks` locally + raw curl + `tar -xzf` in CI. Filename pattern is `betterleaks_<version>_linux_x64.tar.gz` — note the embedded version.
- [Claude Code tool taxonomy migrated 2026 (v2.1.16/v2.1.63)](solutions/api-patterns/claude-code-tool-taxonomy-2026.md) — `Task`→`Agent` rename and `TodoWrite`→`TaskCreate` family split for interactive sessions; split DuckDB views by semantic (launcher vs tracker), keep `TodoWrite` for `--print`/SDK, COALESCE `taskId`/`id` across native + mcp variants.
- [uv-native PyPI publish with isolated smoke tests](solutions/api-patterns/uv-native-pypi-publish-with-smoke-tests.md) — Astral's recommended shape: `uv build` + `uv run --isolated --no-project --with dist/*.whl <pkg> --version` + `uv publish [--index testpypi]`. The smoke step catches missing transitive deps before upload (the wheel METADATA exposes the gap, your dev venv hides it).
- [leidenalg `getattr()` shim for `quality()` / `membership`](solutions/api-patterns/leidenalg-getattr-shim.md) — `CPMVertexPartition` has these accessors at runtime but the type stubs don't expose them; ty + Pyright flag them as unresolved. Type the wrapper as `object` and route attribute access through `getattr(partition, "quality")()` with `# noqa: B009`. Cleanest reversible workaround for leidenalg 0.11 + igraph 1.0.
- [DuckDB `read_json` re-binds schema on every reference](solutions/api-patterns/duckdb-read-json-rebinds-schema-on-describe.md) — every CREATE OR REPLACE VIEW that touches a `read_json(..., union_by_name=true, sample_size=-1)` view re-runs full JSON schema inference; per-view `DESCRIBE` does too. Bypass with a static VIEW_SCHEMA dict + drift test, or replace the DESCRIBE loop with `duckdb_columns()` (1000× faster).
- [`duckdb_functions()` returns null parameters for table macros](solutions/api-patterns/duckdb-table-macro-null-parameters.md) — only scalar macros expose params via the catalog; table macros (`AS TABLE (...)`) come back null. Hand-maintain `MACRO_SIGNATURES` next to `MACRO_NAMES` with a regex drift test that parses the DDL strings.

### best-practices/

- [Sharded cache field-name stability](solutions/best-practices/sharded-cache-field-name-stability.md) — when a path's storage shape shifts (file→dir), reinterpret semantics in a helper, don't rename the Settings field.
- [Anthropic canonical XML tags](solutions/best-practices/anthropic-xml-canonical-tags.md) — use `<instructions>`, `<context>`, `<examples><example>`, `<anti_patterns>`; avoid invented tags like `<task>`.
- [anyio for blocking-IO async pipelines](solutions/best-practices/anyio-structured-concurrency-for-blocking-io.md) — CapacityLimiter + `to_thread.run_sync` propagate cancellation to the thread pool, unlike `asyncio.to_thread`.
- [SARIF scanners: split report vs gate](solutions/best-practices/sarif-scanner-report-vs-gate.md) — run scanner with `|| true` + SARIF → upload → re-run to gate. Decouples code-scanning visibility from CI blocking.
- [Settings derivation via model_validator — never break user pins](solutions/best-practices/team-corpus-glob-via-model-validator.md) — when one Settings field re-derives others, compare against factory output (not literal strings) to detect "user pinned"; mutate via `object.__setattr__`.
- [Stacked PRs need a synchronize event after retarget](solutions/best-practices/branch-protection-pr-targeting-ci-trigger.md) — workflows filter on `branches: [main]`; `gh pr edit --base main` alone doesn't trigger CI; push an empty `ci: trigger` commit.
- [`cz bump` ↔ branch-protected `main`: branch-PR-tag dance](solutions/best-practices/cz-bump-via-pr-with-branch-protection.md) — `mise run bump && git push main` is rejected (GH013); re-bump on a release branch, squash-merge the PR, re-tag the new merge SHA, then push the tag and cut the release.
- [SARIF uploads need a unique `category:` per scanner since 2025-07-22](solutions/best-practices/sarif-category-uniqueness-since-2025-07.md) — GitHub stopped merging same-tool SARIF runs; collisions silently overwrite findings without an error. One unique `category:` per scanner per matrix leg.
- [Importing helpers from `conftest.py` needs both pythonpath + ty root](solutions/best-practices/conftest-import-needs-pythonpath-and-ty-root.md) — `from conftest import FakeFoo` works at runtime only with `[tool.pytest.ini_options].pythonpath = ["tests"]`; static type-check needs ty `root = ["./src", "./tests"]` too.
- [Raw `os.environ[...]` in a test helper leaks across modules](solutions/best-practices/os-environ-leak-across-test-modules.md) — when `monkeypatch` isn't available, pair the assignment with a module-level autouse `_purge_*_env` fixture that snapshots and restores. Otherwise pydantic-settings env-coupled tests in *other* modules silently break.
- [CodeQL flags narrow `except: pass` and unused nested classes](solutions/best-practices/codeql-finds-narrow-except-and-unused-nested-classes.md) — ruff `S110` only fires on broad excepts; ruff `F841` doesn't see classes defined inside a function. Add an explanatory comment to every empty except, delete unused scaffolding, codified in CLAUDE.md § "CodeQL hygiene".
- [Single-logger discipline: ban stdlib `logging` when the package uses loguru](solutions/best-practices/loguru-only-no-stdlib-logging.md) — replace tenacity's `before_sleep_log(stdlib_logger, level)` with a custom `loguru_before_sleep("LEVEL")` callback; enforce the ban via ruff `flake8-tidy-imports.banned-api`. Codified in CLAUDE.md § "Logging: loguru only".
- [PyPI Trusted Publishing requires separate registration on pypi.org AND test.pypi.org](solutions/best-practices/pypi-trusted-publishing-dual-registration.md) — they're separate sites with separate publisher databases; missing TestPyPI registration produces a misleading `invalid-publisher` OIDC error even when claims look correct. Workflow filename in the form is bare (`publish.yml`), not the path.
- [`__version__` should read from `importlib.metadata`, not a hardcoded constant](solutions/best-practices/version-from-importlib-metadata-not-init-constant.md) — `cz bump` (with `version_provider = "uv"`) only updates `[project].version`. A hardcoded `__version__ = "X.Y.Z"` in `__init__.py` drifts and surfaces only in the published wheel's `--version` output. Use `importlib.metadata.version("pkg")` to derive at runtime.
- [`cz changelog` gate needs a release-PR carve-out](solutions/best-practices/cz-changelog-gate-release-pr-carveout.md) — `cz changelog --incremental` exits with "No tag found" on a release branch where CHANGELOG.md already has the new version header but the tag doesn't exist yet. Skip via `!startsWith(github.event.pull_request.title, 'chore(release):')` — title-based, stable across amends, no new convention to maintain.
- [Rebase onto origin/main before `cz bump`](solutions/best-practices/rebase-before-bump.md) — `cz bump` walks local history only; if origin shipped a release while you were branched, your computed version (and tag) will collide with what's already published. 30-second `git fetch && git log main..origin/main` before bumping prevents the rip-and-restart of v0.3.0 → v0.5.0 we just hit on `feat/leiden-cpm`.
- [Parquet-existence gate over log-demotion for optional analytics DDL](solutions/best-practices/parquet-existence-gate-over-log-demotion.md) — when a DDL block depends on optional parquet caches, gate the DDL on `_parquet_is_populated(path)` instead of `try/except + logger.warning`. The gate encodes the invariant; `_safe_macro` (DEBUG, not WARNING) stays as defense-in-depth backstop. Eliminates noisy WARNINGs on fresh-install runs.

### conventions/

- [`tool.uv` `required-version` pin needs mise-shimmed PATH for git hooks](solutions/conventions/uv-version-pin-via-mise.md) — bumping `[tool.uv] required-version` is a silent landmine for any contributor whose interactive shell predates the install: `mise run check` works (mise tasks resolve through their own shim) but `git commit` blows up because the lefthook pre-commit inherits the parent shell's stale PATH and runs the old uv. Quick fix: `export PATH="$HOME/.local/share/mise/installs/uv/<new>/uv-<arch>:$PATH"` before the commit. Persistent: `mise reshim && exec zsh`.
- [`cz changelog` overwrites hand-written CHANGELOG entries](solutions/conventions/changelog-drift-gate-cz-overwrites-handwritten.md) — when CI runs `cz changelog` and asserts no drift, hand-written `### BREAKING CHANGE` body text gets obliterated. Put narrative in the PR description and commit body; let cz regenerate CHANGELOG.md as a follow-up `chore(changelog): regenerate` commit.
