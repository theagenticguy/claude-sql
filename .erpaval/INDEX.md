## By category

### api-patterns/

- [DuckDB list_avg is per-row](solutions/api-patterns/duckdb-list-avg-aggregate.md) — for centroid math use `unnest+generate_subscripts+groupby+list ORDER BY pos`.
- [Bedrock shared client + pool sizing](solutions/api-patterns/bedrock-shared-client-and-pool.md) — one shared boto3 client with `max_pool_connections=max(32, 2×conc)`; default of 10 silently caps concurrency.
- [Anthropic tokenizer gap + cache minimums](solutions/api-patterns/anthropic-prompt-cache-tokenizer-gap.md) — cl100k_base overcounts Anthropic tokens ~22%; Haiku 4.5 cache min is 4096. **2026-05-13 update**: Sonnet 4.6 cache min bumped 1024 → **2048**; AWS Bedrock User Guide is canonical for `ttl="1h"` availability on `global.anthropic.claude-sonnet-4-6` (Anthropic's docs still claim Bedrock doesn't support 1h — stale).
- [Global CRIS not in ListInferenceProfiles](solutions/api-patterns/cris-profile-not-in-list.md) — `global.<vendor>.<model>` may invoke fine even when absent from the profile list; verify by invoke, not listing.
- [ty strict mode is `rules.all = "error"`](solutions/api-patterns/ty-strict-mode-all-error.md) — ty has no `--strict` flag; the canonical strict knob is `[tool.ty.rules] all = "error"` + `error-on-warning`, minus `division-by-zero`.
- [hdbscan cp314 wheel gap](solutions/api-patterns/hdbscan-cp314-wheel-gap.md) — 0.8.42 is the lone Python 3.14 blocker in the scientific-python closure; sdist fallback is unacceptable for `uv tool install` end-users.
- [Lefthook pre-push `HEAD~` fallback](solutions/api-patterns/lefthook-push-gate-fallback.md) — `@{push}` fails with exit 128 on first-push-of-branch; keep the `|| git diff --name-only HEAD~` fallback.
- [Semgrep in CI: container, not deprecated action](solutions/api-patterns/semgrep-ci-container-not-action.md) — `returntocorp/semgrep-action` is deprecated; `semgrep ci` rejects `--config`. Run `semgrep scan` inside the `semgrep/semgrep` container.
- [CycloneDX SBOM for uv projects](solutions/api-patterns/cyclonedx-python-uv-environment.md) — cdxgen + `uv export` don't speak uv.lock; `cyclonedx-py environment .venv` is the working path.
- [Bedrock structured output: `output_config.format` ≠ Converse](solutions/api-patterns/bedrock-output-config-vs-converse.md) — two API shapes with different field paths and refusal vocabularies; stay on InvokeModel + `output_config.format` to match the existing pipeline (refusal stop_reason, prompt caching, adaptive thinking).
- [PR↔transcript binding: trailers + git notes from `prepare-commit-msg`](solutions/api-patterns/git-trailer-and-notes-binding.md) — commit-trailers + `refs/notes/transcripts` is the survivable, host-agnostic primitive; run from `prepare-commit-msg` (not `commit-msg` or `pre-commit`).
- [bandit `-f sarif` needs the `[sarif]` extra](solutions/api-patterns/bandit-sarif-formatter-extras.md) — without `bandit[sarif]`, `-f sarif` raises `ModuleNotFoundError` at runtime, not install time. Pair with `[tool.bandit] skips` aligned 1:1 with ruff's S-ignores.
- [betterleaks isn't on PyPI — Go binary, install via mise `aqua:`](solutions/api-patterns/betterleaks-not-on-pypi.md) — `uv add betterleaks` fails (not a Python package); use `aqua:betterleaks/betterleaks` locally + raw curl + `tar -xzf` in CI. Filename pattern is `betterleaks_<version>_linux_x64.tar.gz` — note the embedded version.
- [Claude Code tool taxonomy migrated 2026 (v2.1.16/v2.1.63)](solutions/api-patterns/claude-code-tool-taxonomy-2026.md) — `Task`→`Agent` rename and `TodoWrite`→`TaskCreate` family split for interactive sessions; split DuckDB views by semantic (launcher vs tracker), keep `TodoWrite` for `--print`/SDK, COALESCE `taskId`/`id` across native + mcp variants.
- [uv-native PyPI publish with isolated smoke tests](solutions/api-patterns/uv-native-pypi-publish-with-smoke-tests.md) — Astral's recommended shape: `uv build` + `uv run --isolated --no-project --with dist/*.whl <pkg> --version` + `uv publish [--index testpypi]`. The smoke step catches missing transitive deps before upload (the wheel METADATA exposes the gap, your dev venv hides it).
- [leidenalg `getattr()` shim for `quality()` / `membership`](solutions/api-patterns/leidenalg-getattr-shim.md) — `CPMVertexPartition` has these accessors at runtime but the type stubs don't expose them; ty + Pyright flag them as unresolved. Type the wrapper as `object` and route attribute access through `getattr(partition, "quality")()` with `# noqa: B009`. Cleanest reversible workaround for leidenalg 0.11 + igraph 1.0.
- [DuckDB `read_json` re-binds schema on every reference](solutions/api-patterns/duckdb-read-json-rebinds-schema-on-describe.md) — every CREATE OR REPLACE VIEW that touches a `read_json(..., union_by_name=true, sample_size=-1)` view re-runs full JSON schema inference; per-view `DESCRIBE` does too. Bypass with a static VIEW_SCHEMA dict + drift test, or replace the DESCRIBE loop with `duckdb_columns()` (1000× faster).
- [`duckdb_functions()` returns null parameters for table macros](solutions/api-patterns/duckdb-table-macro-null-parameters.md) — only scalar macros expose params via the catalog; table macros (`AS TABLE (...)`) come back null. Hand-maintain `MACRO_SIGNATURES` next to `MACRO_NAMES` with a regex drift test that parses the DDL strings.
- [lancedb 0.30 `list_tables()` returns a pydantic model](solutions/api-patterns/lancedb-list-tables-pagination-shape.md) — `ListTablesResponse(tables=[...], page_token=...)` with a typed `.tables: list[str]` field, NOT a flat list and NOT paginated tuples. `db.table_names()` is deprecated and emits warnings. Don't write defensive shape-shims; use `name in db.list_tables().tables`.
- [DuckDB `ATTACH (TYPE LANCE)` is permissive](solutions/api-patterns/duckdb-attach-lance-empty-namespace.md) — succeeds on missing/empty/garbage paths; the catalog error fires later at SELECT/CREATE-VIEW time. Filesystem heuristics like `any(dir.iterdir())` are wrong; probe the LanceDB namespace via `_has_table` instead. **2026-05-13 update**: second `register_vss` after `embed` populates Lance hits "Existing object is of type Table" — `DROP VIEW IF EXISTS` doesn't match a Table, so the rebind helper drops both shapes and `DETACH lance_store` before re-attaching.
- [DuckDB `LAG` over a named window beats correlated-min on `read_json` views](solutions/api-patterns/duckdb-lag-named-window-vs-correlated-min.md) — every textual reference to a `read_json(..., union_by_name=true, sample_size=-1)` view re-runs schema inference; correlated-min has 3 references vs LAG's 1, hence ~3× wall-clock. Use `LAG(...) OVER w` with a single `WINDOW w AS (PARTITION BY ... ORDER BY ts, uuid)` clause for adjacency views like `turn_window`.
- [anyio dispatcher must NOT acquire the CapacityLimiter when the task body already does](solutions/api-patterns/anyio-dispatcher-no-double-limiter.md) — `classify_one` already holds the slot; an outer `async with sem:` at the dispatch layer serializes the entire fan-out. Count the `async with sem:` lines in your worker — there should be exactly one per limiter, at the SDK call boundary.
- [Ruff BLE001 doesn't fire on `except Exception: ...; raise`](solutions/api-patterns/ruff-BLE001-not-fired-on-bare-reraise.md) — adding `# noqa: BLE001` to a bare-reraise wrapper trips RUF100 (unused noqa). Put the rationale comment on the line above the except instead. CodeQL `py/empty-except` is satisfied; ruff stays quiet.
- [uv_build PEP 420 form 1a — explicit dotted `module-name`](solutions/api-patterns/uv-build-namespace-form-1a-explicit-module-name.md) — for namespace packages under uv workspaces, use `[tool.uv.build-backend] module-name = "ns.member"` per member. NOT `namespace = true`, NOT `module-root = ""`. Three alternative forms; only 1a is the recommended shape. The bonk synthesis from 2026-05-14 conflated them.
- [import-linter v2 renamed `layered` → `layers`](solutions/api-patterns/import-linter-v2-contract-type-renames.md) — `type = "layered"` fails silently in v2.x: exit 1, prints contract name, no error visible until `--debug` reveals `NoSuchContractType: layered`. Fix is one character. `independence` and `forbidden` keep their v1 names.
- [`uv sync --all-packages` for virtual-root workspaces](solutions/api-patterns/uv-workspace-virtual-root-needs-all-packages.md) — when workspace root pyproject sets `package = false`, `uv sync` resolves dev-deps but does NOT install members. `--all-packages` is required. Symptom is `ModuleNotFoundError` for every member after a clean `uv sync`. Codify in `mise run install`.

### best-practices/

- [Sharded cache field-name stability](solutions/best-practices/sharded-cache-field-name-stability.md) — when a path's storage shape shifts (file→dir), reinterpret semantics in a helper, don't rename the Settings field.
- [Anthropic canonical XML tags](solutions/best-practices/anthropic-xml-canonical-tags.md) — use `<instructions>`, `<context>`, `<examples><example>`, `<anti_patterns>`; avoid invented tags like `<task>`.
- [anyio for blocking-IO async pipelines](solutions/best-practices/anyio-structured-concurrency-for-blocking-io.md) — CapacityLimiter + `to_thread.run_sync` propagate cancellation to the thread pool, unlike `asyncio.to_thread`.
- [SARIF scanners: split report vs gate](solutions/best-practices/sarif-scanner-report-vs-gate.md) — run scanner with `|| true` + SARIF → upload → re-run to gate. Decouples code-scanning visibility from CI blocking.
- [Settings derivation via model_validator — never break user pins](solutions/best-practices/team-corpus-glob-via-model-validator.md) — when one Settings field re-derives others, compare against factory output (not literal strings) to detect "user pinned"; mutate via `object.__setattr__`.
- [Stacked PRs need a synchronize event after retarget](solutions/best-practices/branch-protection-pr-targeting-ci-trigger.md) — workflows filter on `branches: [main]`; `gh pr edit --base main` alone doesn't trigger CI; push an empty `ci: trigger` commit.
- [`cz bump` ↔ branch-protected `main`: branch-PR-tag dance](solutions/best-practices/cz-bump-via-pr-with-branch-protection.md) — `mise run bump && git push main` is rejected (GH013); re-bump on a release branch, squash-merge the PR, re-tag the new merge SHA, then push the tag and cut the release.
- [`cz bump` hides `chore` commits from CHANGELOG — use `refactor` for deprecation removals](solutions/best-practices/cz-changelog-prefer-refactor-for-deprecation-removals.md) — default cz changelog filter surfaces feat/fix/refactor; chore is invisible. Pick `refactor:` for deletion commits closing a deprecation window so users see the removal. Stays at PATCH.
- [SARIF uploads need a unique `category:` per scanner since 2025-07-22](solutions/best-practices/sarif-category-uniqueness-since-2025-07.md) — GitHub stopped merging same-tool SARIF runs; collisions silently overwrite findings without an error. One unique `category:` per scanner per matrix leg.
- [Importing helpers from `conftest.py` needs both pythonpath + ty root](solutions/best-practices/conftest-import-needs-pythonpath-and-ty-root.md) — `from conftest import FakeFoo` works at runtime only with `[tool.pytest.ini_options].pythonpath = ["tests"]`; static type-check needs ty `root = ["./src", "./tests"]` too.
- [Raw `os.environ[...]` in a test helper leaks across modules](solutions/best-practices/os-environ-leak-across-test-modules.md) — when `monkeypatch` isn't available, pair the assignment with a module-level autouse `_purge_*_env` fixture that snapshots and restores. Otherwise pydantic-settings env-coupled tests in *other* modules silently break.
- [CodeQL flags narrow `except: pass`, unused nested classes, AND `except BaseException` in async dispatchers](solutions/best-practices/codeql-finds-narrow-except-and-unused-nested-classes.md) — ruff `S110`/`F841`/`BLE001` each catch a strict subset; CodeQL is the strict superset. Three patterns: comment every empty except, delete unused scaffolding, narrow `BaseException → Exception` in any anyio task body so `CancelledError` propagates. **2026-05-13 update**: PR #42 added the BaseException finding pattern after `trajectory_worker.py:647/:862` swallowed cancellation in the per-session task group.
- [Single-logger discipline: ban stdlib `logging` when the package uses loguru](solutions/best-practices/loguru-only-no-stdlib-logging.md) — replace tenacity's `before_sleep_log(stdlib_logger, level)` with a custom `loguru_before_sleep("LEVEL")` callback; enforce the ban via ruff `flake8-tidy-imports.banned-api`. Codified in CLAUDE.md § "Logging: loguru only".
- [PyPI Trusted Publishing requires separate registration on pypi.org AND test.pypi.org](solutions/best-practices/pypi-trusted-publishing-dual-registration.md) — they're separate sites with separate publisher databases; missing TestPyPI registration produces a misleading `invalid-publisher` OIDC error even when claims look correct. Workflow filename in the form is bare (`publish.yml`), not the path.
- [`__version__` should read from `importlib.metadata`, not a hardcoded constant](solutions/best-practices/version-from-importlib-metadata-not-init-constant.md) — `cz bump` (with `version_provider = "uv"`) only updates `[project].version`. A hardcoded `__version__ = "X.Y.Z"` in `__init__.py` drifts and surfaces only in the published wheel's `--version` output. Use `importlib.metadata.version("pkg")` to derive at runtime.
- [`cz changelog` gate needs a release-PR carve-out](solutions/best-practices/cz-changelog-gate-release-pr-carveout.md) — `cz changelog --incremental` exits with "No tag found" on a release branch where CHANGELOG.md already has the new version header but the tag doesn't exist yet. Skip via `!startsWith(github.event.pull_request.title, 'chore(release):')` — title-based, stable across amends, no new convention to maintain.
- [Rebase onto origin/main before `cz bump`](solutions/best-practices/rebase-before-bump.md) — `cz bump` walks local history only; if origin shipped a release while you were branched, your computed version (and tag) will collide with what's already published. 30-second `git fetch && git log main..origin/main` before bumping prevents the rip-and-restart of v0.3.0 → v0.5.0 we just hit on `feat/leiden-cpm`.
- [Parquet-existence gate over log-demotion for optional analytics DDL](solutions/best-practices/parquet-existence-gate-over-log-demotion.md) — when a DDL block depends on optional parquet caches, gate the DDL on `_parquet_is_populated(path)` instead of `try/except + logger.warning`. The gate encodes the invariant; `_safe_macro` (DEBUG, not WARNING) stays as defense-in-depth backstop. Eliminates noisy WARNINGs on fresh-install runs.
- [SQLite WAL cold-start writers race on `PRAGMA journal_mode=WAL`](solutions/best-practices/sqlite-wal-cold-start-pragma-race.md) — the WAL transition itself acquires the writer lock. N concurrent fresh-DB connections all run the PRAGMA; the loser raises `database is locked` BEFORE `busy_timeout` can absorb it. Hide `journal_mode` + `synchronous` + `CREATE TABLE` behind the same once-per-(process, path) lock as schema bootstrap; keep `busy_timeout` and `foreign_keys` per-connection.
- [stdlib sqlite3 autocommit + `executemany` fsyncs per row](solutions/best-practices/sqlite-bulk-migration-explicit-begin-commit.md) — `isolation_level=None` makes every `executemany` row its own transaction; on WAL with a 39k-row migration on EFS, the per-row fsyncs hang for many minutes. Wrap bulk inserts in explicit `BEGIN`/`COMMIT` (or switch to `"DEFERRED"`). 39k rows hung → 3.76s on the live corpus.
- [GHAS check-runs duplicate Actions job names — they sit `queued` indefinitely](solutions/best-practices/ghas-queued-checks-vs-actions-jobs.md) — when CI uploads SARIF, GitHub Advanced Security creates a check-run with the same name as the scanner's Apps integration. After a rebase the GHAS check binds to the previous SHA's run; the new SHA's check stays `queued` forever. `mergeStateStatus=UNSTABLE`. When the lowercase Actions job passes, the uppercase GHAS check is safe to bypass with `--admin`.
- [GitHub Actions `codeload.github.com` 429 — rerun, don't diagnose](solutions/best-practices/ci-codeload-429-rerun.md) — transient infrastructure failure during action archive download (`Set up job` step). 3-attempt retry exhausts on a busy CDN. Always rerun via `gh run rerun <id> --failed`; never a real failure.
- [Per-pipeline accumulator key must be passed explicitly, not derived from schema metadata](solutions/best-practices/pipeline-accumulator-explicit-key.md) — registering `_CACHE_STATS["trajectory"]` and looking it up via `schema.get("title")` (= `"TrajectoryArrayResult"`) silently no-ops the entire counter. Thread the pipeline name as an explicit `pipeline: str` kwarg through `classify_one`; never infer identity from incidental data shape.
- [Rip-and-replace parquet schema migrations via marker-column probe](solutions/best-practices/parquet-schema-migration-rip-and-replace.md) — when a worker rewrites its parquet schema, scan shards' `pq.ParquetFile.schema_arrow` for distinctive old-schema column names (e.g. `conflict_idx`, `empty`) at startup and `shutil.rmtree` the cache dir on first hit. Idempotent on subsequent runs. Cheaper-and-simpler than in-place migration for analytics caches that are recomputable from the JSONL corpus.
- [CodeQL `py/file-not-closed` in test mkstemp fakes](solutions/best-practices/codeql-py-file-not-closed-in-test-mkstemp-fakes.md) — when monkeypatching `tempfile.mkstemp` to return a `(fd, path)` tuple, open the fd inside the closure (per-call), not at module level. Per-call open lets the production consumer's existing `os.close(fd)` own the close, satisfying both CodeQL's data-flow tracker AND avoiding the `Bad file descriptor` double-close that an `addFinalizer` shape produces.
- [Checkpointer gates computation but not cache replacement](solutions/best-practices/checkpointer-doesnt-replace-sharded-cache-rows.md) — when a session-level checkpointer admits a growing session for re-scoring, `write_part` appends a fresh shard alongside the stale one; every `(session_id, prev_uuid, curr_uuid)` pair duplicates on rerun. Pair the computation gate with a `parquet_shards.replace_sessions` call inside the same `write_lock` critical section so rerun writes are replace-then-append. GH #45 / PR #54.
- [Read-side dedup companions write-side replace-then-append](solutions/best-practices/read-side-dedup-companions-write-side-replace.md) — for views over sharded parquet caches that promise partition-key uniqueness, pair the writer-side `replace_sessions` with a `QUALIFY row_number() OVER (PARTITION BY <key> ORDER BY <freshness> DESC NULLS LAST) = 1` filter in the view DDL. Belt-and-suspenders. DuckDB's `QUALIFY` keeps the projection clean (no `_rn` to `EXCLUDE`); use a parallel `view_qualify` dict in `register_analytics` so the legacy-schema fallback `SELECT *` path naturally drops the QUALIFY for shards without the freshness column. GH #46.

### conventions/

- [`tool.uv` `required-version` pin needs mise-shimmed PATH for git hooks](solutions/conventions/uv-version-pin-via-mise.md) — bumping `[tool.uv] required-version` is a silent landmine for any contributor whose interactive shell predates the install: `mise run check` works (mise tasks resolve through their own shim) but `git commit` blows up because the lefthook pre-commit inherits the parent shell's stale PATH and runs the old uv. Quick fix: `export PATH="$HOME/.local/share/mise/installs/uv/<new>/uv-<arch>:$PATH"` before the commit. Persistent: `mise reshim && exec zsh`.
- [CHANGELOG.md is write-only via `cz bump`](solutions/conventions/changelog-write-only-via-cz-bump.md) — drop the drift gate. `cz bump` is the only thing that writes to the file; squash-merge appends `(#NNN)`. No `## Unreleased`, no manual edits, no CI byte-diff. The drift gate conflated three things — squash-merge `(#NNN)` injection, hollow `## Unreleased` placeholder, real content drift — and the first two false-positives ate hours/week. Between releases, `mise run bump:dry-run` shows what would land.
- [`cz changelog` overwrites hand-written CHANGELOG entries (SUPERSEDED)](solutions/conventions/changelog-drift-gate-cz-overwrites-handwritten.md) — superseded by `changelog-write-only-via-cz-bump.md`. Old advice was "live with the gate, write narrative in commit body + PR description". The narrative-discipline part is still good; the "live with the gate" framing is wrong now. We deleted the gate.
- [Spiral recovery: ground in installed source, not training data](solutions/conventions/erpaval-spiral-recovery.md) — when caught defending against an unverified API shape, `EnterPlanMode` → Read `.venv/.../<package>/<module>.py` → `uv run python -c` live probe → write down what's actually true → only then plan edits. Cost of pausing: ~10 min. Cost of not pausing: hours of edits-on-edits. Pinned by the lance-store gotchas (`api-patterns/lancedb-list-tables-pagination-shape.md`, `api-patterns/duckdb-attach-lance-empty-namespace.md`).
- [Test fakes for wide-fanout helpers absorb future kwargs (`*_a, **_kw`)](solutions/conventions/test-fakes-absorb-future-kwargs.md) — when a fake monkeypatches a helper called from 3+ production sites (e.g. `classify_one`), default to `*_a: object, **_kw: object` so future kwargs slot through silently. One production-side signature change should not cascade to N test-file edits. Use tight signatures only when the test asserts on the kwarg shape.
- [ERPAVal parallel agents on a shared file: partition by symbol, or serialize](solutions/conventions/erpaval-parallel-agents-shared-file.md) — when two Act subagents both edit `cli.py` concurrently, the smaller commit can silently revert the larger one (last-write-wins via git). Either name the symbols/line-ranges each agent owns in the task packet, OR dispatch the wave serially when symbols overlap textually. Agent-side discipline: `git fetch && rebase` before the final commit.
- [Parallel coverage agents driven by exact uncovered line lists](solutions/conventions/parallel-coverage-agents-targeted-uncovered-lines.md) — codecov-patch failures don't lie about which lines are missing; spawn one Agent per file with the verbatim ``--cov-report=term-missing`` output, a pointer to the existing test file (for fixture imitation), CLAUDE.md context, and a single-file write target. Three parallel agents on PR #42 lifted patch coverage 82.78% → 91%+ in ~20 min wall-clock; serial would have been ~40 min.
- [Sed rename pass — three categories of misses](solutions/conventions/sed-rename-pass-three-categories-of-misses.md) — a line-anchored sed pass to rewrite imports during a module rename catches the dominant `^from claude_sql.<X> import` form. Three categories slip past: (1) line-mid lazy imports inside function bodies, (2) multi-name `from claude_sql import (a, b, c)` blocks where names span clusters, (3) string-form module references (`"claude_sql.X" in sys.modules`, `python -m claude_sql.X`, `monkeypatch.setattr("claude_sql.X.boto3.client", ...)`. Plan for each. Run pytest collect + ty check + runtime imports as the discovery loop.
